# 🛒 Order Management System (OMS)

A full-stack **Customer Support Dashboard** built with **Java Spring Boot 3.2**, **MySQL 8**, and an integrated **Python FastAPI ML service** for order risk scoring.  
Supports secure authentication, real-time order management, customer profiles, order notes, IT support ticketing, and machine‑learning powered risk insights. [web:83][web:84][web:52]

---

## 🔧 Tech Stack

| Layer        | Technology                                         |
|-------------|----------------------------------------------------|
| Frontend    | HTML5, CSS3, Vanilla JavaScript (ES6+)             |
| Backend     | Java 22, Spring Boot 3.2                           |
| ML Service  | Python 3, FastAPI, scikit-learn (RandomForest)     |
| Database    | MySQL 8, Hibernate JPA                             |
| API         | REST (JSON)                                        |
| Security    | Spring Security 6, JWT Authentication              |

## ✨ Features

- 🔐 **Secure JWT login** with role-based access control (CSR dashboard behind auth). [web:88][web:91]
- 📊 **Live order dashboard** backed by MySQL and exposed via REST APIs.
- 📈 **Stat cards** showing real-time Failed / Declined / Canceled / Completed counts.
- 🔍 **Smart search & filtering** by order ID, customer, store, and status.
- 👁️ **Order Detail Modal** with line items, tax breakdown, fulfillment, IT flags, and notes.
- 🧑 **Customer Detail Modal** with profile info and full order history.
- ✏️ **Order notes system** with note types (Internal / Customer / System) and user attribution.
- 🛠️ **Modify orders UI** with live total recalculation and tax updates.
- 🎫 **IT support ticketing** attached to orders, including priority and category.
- 🤖 **ML risk score badge** per order, generated by a FastAPI + RandomForest model and surfaced in the UI. [web:52][web:83]
- 📱 **Responsive layout** optimized for desktop support workflows.

## 🚀 Getting Started

### Prerequisites

- Java 22+ (SDKMAN, Zulu, or Temurin)
- Python 3.10+ (for the ML service)
- MySQL 8+ (local install or Docker)
- Maven 3.9+
- IntelliJ IDEA (recommended) and a Python IDE/venv

---

### ⚡ Quick Setup

#### 1. Clone the repository

```bash
git clone https://github.com/lengtran/order-management-java.git
cd order-management-java
```

#### 2. Create MySQL database

```bash
mysql -u root -p -e "CREATE DATABASE oms_db CHARACTER SET utf8mb4;"
```

#### 3. Configure Spring Boot

```bash
cp src/main/resources/application.properties.example \
   src/main/resources/application.properties
```

Update `application.properties` with your MySQL URL, username, and password.

#### 4. Run the Spring Boot application

```bash
mvn spring-boot:run
```

The Java backend and static frontend will be available at:

```text
http://localhost:8080
```

**Default Login**

- Username: `admin`
- Password: `admin123`

#### 5. Start the FastAPI ML service (optional but recommended)

From the ML service directory (for example `ml-service/`):

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8081
```

The ML API docs will be available at:

```text
http://localhost:8081/docs
```

Spring Boot will call this service to fetch per‑order risk scores. [web:52][web:83]

## 🔍 Key API Endpoints

### Auth

| Method | Endpoint           | Description          |
|--------|--------------------|----------------------|
| POST   | `/api/auth/login`  | Generate JWT token   |

### Orders & Customers

| Method | Endpoint                   | Description                  |
|--------|----------------------------|------------------------------|
| GET    | `/api/orders`              | Get list of orders           |
| GET    | `/api/orders/{id}`         | Get order details            |
| GET    | `/api/customers/{id}`      | Get customer profile         |
| GET    | `/api/order-items/{id}`    | Get items for a given order  |
| GET    | `/api/order-notes/{id}`    | Get notes for an order       |
| POST   | `/api/order-notes`         | Add a note to an order       |

### IT Support & ML Risk

| Method | Endpoint                       | Description                         |
|--------|--------------------------------|-------------------------------------|
| POST   | `/api/tickets`                | Create IT support ticket            |
| GET    | `/api/orders/{id}/risk`       | Get ML risk score for an order      |

These endpoints are consumed by the JavaScript dashboard (`app.js`) and, for `/api/orders/{id}/risk`, proxied to the FastAPI ML service. [web:52][web:84]
